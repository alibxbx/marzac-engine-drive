# Couche marzac/

Adaptations d'ECC à notre système. Trois fichiers, chacun reliant un thème fort d'ECC à une décision déjà prise pour le board Alpha de StayLabs.

- `model-routing.md` : quel modèle pour quel agent, et comment garantir l'optimisation des coûts malgré le bug d'héritage.
- `verification-grounding.md` : comment l'auditeur vérifie le contenu, calé sur nos règles éditoriales non-négociables.
- `subagent-retrieval.md` : comment limiter le contexte que chaque sous-agent aspire, pour moins de tokens et plus de focus.

Ces trois leviers se renforcent : router par modèle, vérifier sans inventer, et ne récupérer que le nécessaire.
